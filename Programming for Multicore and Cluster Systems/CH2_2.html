2.5 Interconnection Networks

  • Def : physical connection between different components of parallel systems (internally consists of links and switches)
  • For Multicomputer Systems : connect processors/nodes
    • coordination
    • synchronization
    • exchange of data
  • For Multiprocessor Systems : connect processors with memory modules
    • memory access
  • Design Criteria :
    • Topology : geometric interconnection structure
      • Static Interconnection Network (direct network):
        • direct connection of processors or memory modules
        • fixed physical links
        • used for distribute address space, i.e., nodes-memory modules
      • Dynamic Interconnection Network :
        • switches & links
        • used for both distributed/shared address space, i.e., bus-based network or switching network
    • Routing Technique : exact message transmission used
      • Routing Algorithm : which path to be used for transmission
      • Switching Technology : how a message cut into pieces; how a routing path is assigned to message; how a message is forward
2.5.1 Properties of Interconnection Networks
  • Undirected connection graph :
    • G = (V, E)
    • V : set of nodes
    • E : set of directed connection links
    • Path : a sequence directed connected nodes
  • Static networks Properties :
    • Diameter D(G) : maximum distance between any pair of nodes
      • small diameters lead to small distance
    • Degree g(G) : maximum degree of a node in G
      • small degree reduce hardware overhead
    • Bisection bandwidth B(G) : minimum of edges must be removed to partition the graph into two parts of equal size
      • B(G) + 1 messages can saturate a network
      • measure of capacity of simultaneous message transmission
    • Node/Edge connectivity nc(G)/ne(G) : minimum nodes/edges must fail to disconnect a network
      • availability and reliability
      • minimum g(G) is an upper bound of nc/ne
    • Embedding (G’ to G):
      • assign each node of G’ to a node of G such that:
        • different nodes of G’ are assigned to different nodes of G
        • edges between two nodes in G’ also present between associated nodes in G
      • reflect the flexibility of network
2.5.2 Directed Interconnection Networks

  • Complete Graph :

    • all other networks can be embedded into this network
  • Linear Array Network :
    • Can be embedded into any other network except for tree network
  • Ring Network :
    • In practice, ring network are used for small number of processors as a part of more complex network
  • D-dimensional Mesh :
    • Symmetric mesh : mesh with same extension in all directions
    • the extension in different direction might be different
    • ec = nc = d (for the corner node)
  • D-dimensional Torus :
    • a variation of d-dimensional mesh (additional edges between the first and the last node in each dimension)
    • ec = nc = 2d
  • K-dimensional Cubes :
    • Hamming distance
    • diameter = degree = ec = nc = k
    • bisection = n/2 (n=2^k)
    • there are k independent path between any two nodes v and u:
      • do not share any edge
      • only share two nodes v and u
    • allows embedding of :
      • cube-connected cycles
      • complete binary tree
      • k-dimensional shuffle-exchange network
      • k-ary d-cube
  • Cube-connected Cycles (CCC) :
    • from k-dimensional cube, replace each node with a cycle of k nodes
    • degree = 3
    • ec = nc = 3
    • diameter = 2k - 1 + k/2 (the lower bound)
    • Bisection = n/(2k), where n=k*2^k
  • Complete Binary Tree :
    • k level tree
    • number of nodes = 2^k - 1
    • degree = 3
    • diameter = 2 * log (n/2) or 2*(k-1)
    • ne = nc = 1
    • bisection = 1
  • K-dimensional shuffle-exchange network :
    • number of nodes = 2^k
    • number of edges = 3*2^(k-1)
      • nodes differs in the last bit — exchange edge
      • nodes results from right/left cyclic shift — shuffle edge
    • perfect shuffle — cyclic left shift
    • inverse perfect shuffle — cyclic right shift
  • k-ary d-cube :
    • k = 2 : each node has one neighbor in each dimension
    • k > 2 : each node has two neighbor in each dimension
    • k-ary 2-cube is torus with k^2 nodes
    • 3-ary 3-cube is 3-d torus with 3^3 nodes
    • 2-ary d-cube is d-dimensional cube
2.5.3 Embedding
  • Reflected Gray code (RGC) : neighboring strings differs in only one bit

Embedding Ring into Hypercube

  • k-bit RGC is a bijection function from 2^k ring network to k-d cube

Embedding Two-Dimensional Mesh into Hypercube

  • 2-d mesh with n = n1 * n2 nodes
  • k-d cube with n = 2^k nodes
  • let n1 = 2^k1; n2 = 2^k2
  • denote nodes in 2-d mesh with 2 RGC of k1 and k2 bits
  • concatenation of two RGC code is the bijection mapping
  • embedding d-dimensional mesh is the same method
2.5.4 Dynamic Interconnection Network
  • switches are used to provide an indirect connection between nodes
  • switches are connected by physical links
  • dynamic networks can be characterized according to their topological structure (e.g. bus network, multistage network, crossbar network)
2.5.4.1 Bus network
  • graph :
  • at each point of time, only one data transport can be performed via the bus
  • bus arbiter is used to coordinate the simultaneous transportation
  • bus network are typically used for a small number of processors
2.5.4.2 Crossbar network
  • structure:
    • for system with shared memory space, inputs are processors and outputs are memory modules
    • for system with distribute memory space, both input and output are processors
  • crossbar network are only used for small number of processors because of large hardware overhead
2.5.4.3 Multistage switching networks
  • structure :
  • the goal is to obtain a small distance for arbitrary pairs of inputs and outputs devices to communicate
  • Regular multistage interconnection networks:
    • DEF :
      • regular construction method by using same degree of incoming and outgoing wires for all switches
      • neighbor stages are connected by fixed interconnections
    • graph property of regular multistage:
      • connection between neighboring stages are directed acyclic graph of depth 1
      • degree of each node is g = n/w (where n is number of edges between neighboring stages, w nodes for each stage)
      • connection between neighboring stages is {1, 2, …, n} -> {π(1), π(2), …, π(n)}
      • same permutation π is used for all stages, and stage number can be used as a parameter
    • 2 * 2 crossbar switches :
      • 2*2 crossbar switches result in log(n) stages
2.5.4.4 Omega Network
  • n * n omega network ((logn - 1)-dimensional omega network) based on 2 * 2 crossbar network in log(n) stages
  • each stage has n/2 switches
  • (n/2) * log(n) switches
  • Permutation function :
    • (a, i) represent switch within stage i
    • a is bit string with length of log(n)
    • (a, i) is linked to two nodes (b, i+1) by
      • b results from a by a cyclic left shift
      • b results from a by a cyclic left shift followed by an inversion of last bit

2.5.4.5 Butterfly Network
  • k-dimensional butterfly network connects n = 2^(k+1) inputs to n = 2^(k+1) outputs using 2*2 crossbar switches
  • k+1 stages
  • 2^k switches per stage
  • Permutation function :
    • (a, i) represent the switch (where a is bit string of length k)
    • (a, i) and (b, i+1) are connected if and only if:
      • a, b are identical
      • a, b differ in precisely the (i+1)th bit from the left
    • This permutation function can be calculated in a recursive way

2.5.4.6 Baseline Network
  • k-dimensioanl baseline network
  • number of nodes and switches are the same as butterfly network
  • Permutation function :
    • (a, i) is connected to (b, i+1) if and only if
      • b results from a by a cyclic right shift on the last k-i bit of a
      • b results from a by first inverting the rightmost bit from a and then cyclic right shift on the last k-i bits (from right)
    • each time confirms one bit from left

2.5.4.7 Benes Network
  • k-dimensional Benes Network is constructed from two k-d butterfly networks
  • 2k+1 stages
  • 2^k switches at each stage
  • Benes Network is able to provide n paths for any two nodes communication
