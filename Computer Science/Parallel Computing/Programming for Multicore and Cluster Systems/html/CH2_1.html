2.1 Processor Architecture and Technology Trends

Increase in number of transistors :
Moore’s Law : number of transistor in a typical processor chip doubles every 18-24 months; 400-800 million transistors on one processor chip in 2009

Increase in clock speed :
Clock Speed : main characterization of the performance of a computer system; SPEC benchmarks (System Performance and Evaluation Cooperative) are used to measure the integer and floating-point performance of computer systems.

Four phases of processor design trends driven by internal use of parallelism

  • Parallelism at Bit Level :

    • driven by demand for improved floating-point accuracy and a larger address space
    • increasing of Word Size from 4 bits to 64 bits for floating-point accuracy and large space address
    • this trend has slowed down and ended with the adoption of 64-bits operations beginning in the 1990s, since this gives sufficient accuracy for floating-point numbers and covers a sufficiently large address space of 2^64 bytes.

  • Parallelism by Pipelining : (at instruction level)

    • overlapping execution of multiple instructions, instruction are partitioned into several steps (corresponding to delicate hardware units)

      • fetch : fetch next instruction from memory
      • decode :
      • execute : load operands and execute instruction
      • write-back : write result into target register

    • Throughput : the number of instructions finished per time unit (without dependencies, the throughput is one instruction per clock cycle)

    • Degree of parallelism (of pipelined computation) :
      • determined by the number of pipeline stages (without instruction and data dependencies which might cause threads context switching or CPU flushing thus reduce the parallelism)

The number of stages cannot be arbitrarily increased since it is not possible to partition the execution of the instruction into a very large number of steps of equal size. Data dependencies inhibits a completely parallel use of the stages

ILP Processors (instruction level parallelism): processors which use pipelining to execute instructions (typically 2-26 stages)

Superpipelined : processors with larger number of pipeline stages

  • Parallelism by Multiple Functional Units : (multiple issue processors)
    • multiple independent function units work in parallel :
      • ALUs (arithmetic logical units)
      • FPUs (floating-point units)
      • load/store units
      • branch units
    • Restrictions :
      • data dependencies between neighboring instructions and control flow
      • ie, superscalar, dispatching instructions to units are determined at runtime by hardware; this may increase the complexity of the circuits significantly
      • by simulation, over four functional units provides little additional gain

Multiple-issue processors :
Superscalar processors
VLIW processors (very long instruction word)

  • Parallelism at Process or Thread Level :
    • the limit of degree of parallelism by pipelining and multiple function units has been reached
    • alternative approach is to put multiple independent processor cores onto a single processor chip : Multicore Processors
    • each core has a separate flow of control
    • the cores access shared memory

2.2 Flynn’s Taxonomy of Parallel Architectures

  • Flynn’s taxonomy characterize parallel computers according to
    • global control
    • data flows
    • control flows
    • program flows
  • Single-Instruction, Single Data (SISD)
  • Multiple-Instruction, Single-Data (MISD)
    • multiple processing elements with private program memory
    • one common access to single global data memory
  • Single-Instruction, Multiple-Data (SIMD)
    • multiple processing elements with private access to shared/distributed data memory
    • only one program memory, a special control processor fetch and dispatch instructions; each processing element get the same instructions from control processor
    • efficient for applications with significant degree of data parallelism (i.e., multimedia, graphics)
  • Multiple-Instruction, Multiple-Data (MIMD)
    • multiple processing elements with separate instruction and data access to shared/distributed program/data memory
    • processing elements works asynchronously
    • i.e., multicore or cluster systems

2.3 Memory Organization of Parallel Computers

  • Physical Memory Organization :
    • shared memory (multiprocess)
    • distributed memory (multicomputers)
  • For the programmer’s point of view :
    • with distributed address space
    • with a shared address space
2.3.1 Computers with Distributed Memory Organization
  • Distributed Memory Machines (DMM) : computers with physically distributed memory
    • Nodes : processing elements
      • processor
      • local memory : private, only accessible by local processor
    • Interconnection network :
      • connects node
      • support the transfer of data between nodes
    • Examples of interconnection network :
  • DMA Controller (direct memory access) :
    • used to decouple execution of Communication Operations and Processor’s Operation
    • control data transfer between the local memory and the I/O controller (without participation of processor)
    • asynchronous communication :
      • a processor can issue a send operation to the DMA and then continue the local operations while the DMA executes the send operation
      • DMA copies the received message to system location in local memory, when processor performs a receive operation, the data is copied from system location to specified received buffer
  • Routers :
    • used for further decoupling of message forwarding
    • reduce time for non-neighboring communication (do not need the interaction of the processors along the preselected path)
    • each I/O channel of a router can be used for one message at a point of time
    • message buffer are used at each I/O channel
  • Clusters :
    • collection of complete computers with dedicated interconnection network
    • based on standard computers and standard network topologies
    • use standard high-speed interconnection : FCS (Fiber channel standard); SCI (Scalable Coherent Interface)
    • Difference to distributed systems :
      • clusters use same OS and can usually not be addressed individually (job scheduler is used)
  • Grid System :
    • connecting several cluster systems with middleware software (Globus Toolkit)
    • the execution of application programs is controlled by the middleware software
2.3.2 Computers with Shared Memory Organization
  • Shared Memory Machines (SMM) :
    • a number of processors/cores
    • shared physical memory
    • interconnection network
  • Evaluation of global memory :
    • communication via shared vars is easy
    • no data replication is necessary
    • scaling beyond a few dozen processors is difficult (since interconnection network must provide fast access to the global memory for each processor)

Symmetric Multiprocessors (SMPs) a variation of SMM :

  • a single shared memory with uniform access time for all processors and for all memory locations
  • small number of processors with local cache
    • maximum number of processors of a bus-based SMPs is 32-64 (since the central bus provides a constant bandwidth which is shared by all processors)
  • no private memory of processors or specific I/O processors, each processor has a private cache hierarchy
  • cache coherence protocols are used
  • UMAs (uniform memory access)
  • SMMs programs are based on execution of threads
    • Threads : separate control flow, share data with other threads via global address space
      • Kernel Threads : managed by OS; mapped by OS to processors for execution
      • User Threads : explicitly generated and controlled by parallel programs; mapped to kernel threads for execution
  • Distributed Shared Memory Architecture (DSM) :
    • use interconnection network connecting processors of different SMPs
    • use coherence protocol for the view of shared address space
    • NUMAs (non-uniform memory access)
2.3.3 Reducing Memory Access Times (2 approaches)
  • Multithreading (simulating a fixed number of virtual processors for each physical processors)
    • Physical Processor :
      • separate PC (Program Counter)
      • separate sets of registers
      • implicit switch of virtual processes
    • Fine-grained Multithreading : switch is performed after each instruction
      • Drawback 1 : the programming must based on large number of virtual processors, application must have large potential of parallelism to employ all these virtual processors
      • Drawback 2 : physical processors must be specially designed for simulation of virtual processors (software based simulation using standard microprocessors is too slow)
    • Coarse-grained Multithreading : switch only on costly stalls (i.e., level 2 cache miss)
  • Caches :
    • data store in cache is a subset of data in main memory
    • management of data elements in cache is done by hardware (i.e., employing set-associative strategy)
    • access time :
      • L1 : 2-4 cycles
      • Main memory : hundreds cycles
    • employing cache coherence protocol

2.4 Thread-Level Parallelism

  • Def : the architectural organization with in a processor chip require the use of explicitly parallel programs
  • Chip Multiprocessing (CMP) : refers to the corresponding architectural organization (an example is multicore processors)
  • Multicore processors : placement of multiple independent execution cores with all execution resources onto a single processor chip (as an example of CMP)
  • Multithreading :
    • timeslice multithreading
    • switch-on-event multithreading
    • and the following hyper threading technique (i.e., SMT)
2.4.1 Simultaneous Multithreading (SMT)
  • Def : use several threads and schedule executable instructions from different threads in the same cycle, thus using the functional-units more effectively
  • for hardware support :
    • replication of chip area used to store processor state
      • Program Counter
      • Control Register
      • Interrupt controller with register
    • shared resources :
      • caches
      • bus systems
      • control units
      • function units
    • leading to small chip size increase
    • the shared resources are assigned to the logical processors for simultaneous use (leading to simultaneous execution of logical processors)
    • when one logical processor waiting for an event, the shared resources can be assigned to another logical processor
  • for programmer’s view :
    • processor appeared to be a set of logical processors
    • each logical processor stores its processor state in a separate processor resource (avoid overhead for saving/restoring processor states when switching)
    • Waiting time for logical processors :
      • cache misses
      • wrong branch predictions
      • dependencies between instructions
      • pipeline hazards
    • necessary that each logical processors have a separate thread available for execution (application must apply parallel programming to achieve performance for SMT processors)
      SMT is restricted to a small number of logical processors
2.4.2 Multicore Processors
  • Limitations of old technologies in processor performance increasing :
    • Limit of increasing function units :
      • a single control-flow cannot provide enough instruction-level parallelism to keep large amount of function units busy
    • Limit of increasing speed clock :
      • increasing in density of transistors increases the power density and heat production
      • memory access cannot be reduced at the same rate as clock cycles
    • Limit of increasing number of transistors :
      • lead to increase in processor-internal wire length (between functional units)
      • speed of signal transfer with in the wire becomes limit factor

Memory Wall : Processor-to-memory gap due to physical size of processor chips limiting the number of pins on chip, thus limiting bandwidth between processor and main memory

  • Multicore Processor :
    • integrate multiple independent processing core onto one chip
    • switch off unused processor core to reduce energy consumption
    • each execution core represents an independent logical processor
2.4.3 Architecture of Multicore Processors
  • Hierarchical Design
    • multiple core share multiple caches
    • tree-like configuration of caches
    • usage :
      • SMP configuration
      • standard desktop/server processor
      • GPU (Nvidia GeForce 8000 : 128 stream processor (SP) organize in 8 texture/processor clusters (TPC))
    • advantage :
      • scalable
  • Pipelined Designs
    • data elements are processed by multiple execution cores in a pipelined way (from input port to output port)
    • usage :
      • network processors in routers (Xelerator X11 processors)
      • graphics processors

Xelerator X11 contains 800 separate cores arranged in logically linear pipeline

  • Network-Based Design
    • the core of processor chip with local cache/memory are connected via interconnection network with other cores of the chip
    • this network provide support for the synchronization of the cores
  • Challenge of Interconnection :
    • scalable
    • robust to tolerate failure of cores
    • support energy management (i.e., scale-down of power for core by reducing clock speed)
    • I/O system must be able to provide enough bandwidth to keep all cores busy (able to bring hundreds of gigabits per second onto the chip)
